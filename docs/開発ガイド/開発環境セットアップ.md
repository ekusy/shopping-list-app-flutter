# 開発環境セットアップ

このドキュメントは、shopping-list-app-flutter リポジトリを初めて動かす開発者向けのセットアップ手順です。

開発環境は **Docker に統一** しています。ホストマシンに Flutter / Dart / Firebase CLI を直接インストールする必要はありません。すべてのコマンドはプロジェクトルートから `docker compose run` 経由で実行します。

## 前提ツール

| ツール | バージョン / 補足 | 用途 |
|---|---|---|
| Docker | 最新版推奨 | コンテナ実行基盤 |
| Docker Compose | V2（`docker compose`） | サービス定義・コンテナ管理 |
| ブラウザ | ホスト側のもの | Web アプリの表示・デバッグ |

> コンテナ内には **Flutter 3.44 / Dart 3.12 / Firebase CLI** が同梱されている。ホストへの個別インストールは不要。
> ターゲットは **Web のみ**（Android / iOS はコンテナ外の SDK / Xcode が別途必要。詳細は `docs/REMAINING_TASKS.md` を参照）。

---

## 1. 初回セットアップ

### 1-1. クローン

```bash
git clone <repo-url>
cd shopping-list-app-flutter
```

### 1-2. Docker イメージのビルド（初回 / Dockerfile 変更時）

```bash
docker compose build
```

初回は数分かかる。pub パッケージは named volume（`pub_cache`）にキャッシュされ、以降のビルドが高速化される。

### 1-3. 依存解決

```bash
docker compose run --rm flutter flutter pub get
```

### 1-4. Firebase プロジェクトの準備

1. [Firebase コンソール](https://console.firebase.google.com/) でプロジェクトを作成する
2. Authentication を有効化（メール / パスワード方式）
3. Firestore Database を作成（本番モードまたはテストモード）
4. Storage を有効化
5. ウェブアプリを登録し、SDK 設定を取得する

### 1-5. Firebase 接続設定（`firebase_options.dart`）

`lib/firebase_options.dart` はプレースホルダのみのテンプレート。実際の設定値に差し替える。

- 推奨: コンテナ内で FlutterFire CLI を実行して自動生成する。

```bash
docker compose run --rm flutter bash
# コンテナ内で
firebase login --no-localhost
flutterfire configure
```

- もしくは Firebase コンソールで取得した値（apiKey / appId / projectId / messagingSenderId / storageBucket / authDomain）を `firebase_options.dart` の各プラットフォーム定義に手で記入する。

> `firebase_options.dart` の実値・`google-services.json`・`GoogleService-Info.plist` はコミットしない（`.gitignore` 済み）。シークレットは絶対にコミットしないこと。

## 2. アプリの起動（Web）

### 開発サーバー起動（ホットリロード有効）

```bash
docker compose run --rm --service-ports flutter \
  flutter run -d web-server --web-hostname=0.0.0.0 --web-port=5000
```

ホストのブラウザで `http://localhost:5000` にアクセスする。

| キー | 操作 |
|---|---|
| `r` | ホットリロード |
| `R` | フルリスタート |
| `q` | 終了 |

> `--service-ports` を付けないとコンテナのポートがホストに公開されない点に注意。

## 3. テスト実行

```bash
docker compose run --rm flutter flutter test                    # 全テスト実行
docker compose run --rm flutter flutter test test/widgets/      # ウィジェットテストのみ
docker compose run --rm flutter flutter test --name "テスト名"   # テスト名でフィルタ
docker compose run --rm flutter flutter test --coverage         # カバレッジ収集（coverage/lcov.info）
```

テスト構成と方針:

| ディレクトリ | 対象 |
|---|---|
| `test/core/` | `AppError`、`invite_code` / `invite_url`、`item_icons` |
| `test/data/` | リポジトリ実装テスト（`fake_cloud_firestore` 使用） |
| `test/domain/` | エンティティロジック |
| `test/helpers/` | 共通テストユーティリティ（`test_localization.dart`） |
| `test/presentation/` | provider / controller テスト |
| `test/widgets/` | ウィジェットテスト |

> ウィジェットテストで `easy_localization` を使う場合は、必ず `test/helpers/test_localization.dart` の `pumpLocalized` / `setUpTestLocalization`（同期アセットローダー）を使う。標準の `RootBundleAssetLoader` は非同期のため、同一ファイル内の 2 件目以降のテストが空描画になる既知の問題がある。
> ボタン等のタップ対象は `find.bySemanticsLabel` で特定するのが望ましい（`find.text` は絵文字アイコン等で不安定になりやすい）。

## 4. 静的解析・フォーマット

```bash
docker compose run --rm flutter flutter analyze                                         # 静的解析（flutter_lints）
docker compose run --rm flutter dart format lib test                                    # フォーマット（上書き）
docker compose run --rm flutter dart format --output=none --set-exit-if-changed lib test # フォーマットチェックのみ（CI 用）
```

コミット前に `flutter analyze` とフォーマットチェック・テストを通すこと。

## 5. Web 向けビルド & デプロイ（Firebase Hosting）

### ローカルビルド確認

```bash
docker compose run --rm flutter flutter build web --release
# build/web/ に静的ファイルが出力される（named volume: build_vol）
```

ビルド成果物をホストへ取り出す場合:

```bash
id=$(docker create $(docker compose images -q flutter))
docker cp "$id":/app/build/web ./build_output
docker rm "$id"
```

### Firebase Hosting へのデプロイ

```bash
docker compose run --rm flutter firebase deploy --only hosting
```

> 初回は `docker compose run --rm flutter bash` でコンテナに入り、`firebase login --no-localhost` で認証してから実行する。

## 6. Firestore Security Rules のデプロイ・確認

```bash
# Firestore セキュリティルールをデプロイ
docker compose run --rm flutter firebase deploy --only firestore:rules

# ローカルエミュレータ起動（Firestore / Auth）
docker compose run --rm --service-ports flutter firebase emulators:start
```

`firestore.rules` の内容・方針は [`docs/内部設計/アーキテクチャ概要.md` §5](../内部設計/アーキテクチャ概要.md#5-firestore-security-rules-方針) を参照。
リポジトリ層のアクセス挙動は `test/data/` の `fake_cloud_firestore` テストで検証する。

## 10. トラブルシューティング

### ビルドキャッシュが壊れた場合

```bash
docker compose run --rm flutter flutter clean
docker compose run --rm flutter flutter pub get
```

### コンテナを作り直したい場合

```bash
docker compose down
docker compose build
```

### `http://localhost:5000` に接続できない場合

`flutter run` に `--service-ports` と `--web-hostname=0.0.0.0` が付いているか確認する。コンテナ内 `localhost` バインドだとホストから到達できない。

### Firebase が動かない場合

`lib/firebase_options.dart` のプレースホルダが実値に差し替わっているか確認する（`1-5` 参照）。`flutterfire configure` で再生成するのが確実。

### 対話操作したい場合

```bash
docker compose run --rm flutter bash
```
