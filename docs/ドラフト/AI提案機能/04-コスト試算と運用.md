# 04. コスト試算と運用（ドラフト）

> 単価は執筆時点（2026-06）の知識に基づく概算。**実装前に必ず公式料金表で再確認**し、
> 本書の数値を更新すること。

## 1. 前提条件（試算モデル）

| パラメータ | 値 |
|---|---|
| アクティブグループ数 | 100（小規模想定。1,000 でも線形にスケール） |
| 1 グループの平均アクティブアイテム | 30 件 |
| 1 グループの週間イベント（購入+削除） | 30 件 |
| 週次 Gemini 入力 / 出力トークン | 約 2,000 / 約 500 |
| モデル | Gemini Flash 系（[02](./02-週次提案パイプライン設計.md) §4） |

## 2. 月額コスト概算

| 項目 | 計算 | 月額目安 |
|---|---|---|
| Gemini（Vertex AI） | 100 グループ × 4.3 週 × (2K 入力 + 0.5K 出力)。Flash 系単価（入力 ~$0.3/M・出力 ~$2.5/M トークン想定） | **$1 未満** |
| Firestore 書き込み（履歴トリガー） | 30 イベント/週 × 2 書き込み（イベント+サマリー） × 100 グループ × 4.3 週 ≒ 26K 書き込み | 無料枠内〜数十円 |
| Firestore 読み取り（週次パイプライン） | (30 アイテム + 30 サマリー + 100 イベント) × 100 グループ × 4.3 週 ≒ 70K 読み取り | 無料枠内〜数十円 |
| Cloud Functions | トリガー実行 ~13K 回/月 + 週次ジョブ。無料枠 200 万回/月 | 実質ゼロ |
| Cloud Scheduler | `onSchedule` 1 ジョブ（アカウント 3 ジョブまで無料） | ゼロ |
| Firestore TTL 削除 / FCM | 課金対象外 | ゼロ |
| **合計** | | **月数百円未満**（1,000 グループでも月数千円規模） |

ポイント:

- 支配的なコスト項目が存在しない。これは [01](./01-履歴データ設計.md) の
  インクリメンタル集約により「生データ全件を Gemini に渡す」事態を構造的に
  回避しているため。元構成案が最重要視していた「Gemini トークン数の抑制」は
  データ設計の段階で解決済み。
- 休眠グループのスキップ（[02](./02-週次提案パイプライン設計.md) §2.1）により、
  実際の Gemini 呼び出し数はアクティブグループ数に比例する。

## 3. Blaze プラン移行（前提条件）

Cloud Functions / Vertex AI は Spark（無料）プランでは使えないため、
**Phase 0 着手前に Blaze（従量課金）への移行が必須**。

移行時に必ずセットで行うこと:

1. **Cloud Billing 予算アラート**: 月額予算（例: ¥1,000）に対し 50% / 90% / 100% で
   メール通知を設定。
2. **Vertex AI のクォータ確認**: デフォルトのリクエスト/分クォータで週次バッチには十分
   だが、念のため確認。
3. Blaze 移行で Firestore 等の無料枠は維持される（超過分のみ課金）ことの理解。

## 4. 監視・運用

| 仕組み | 内容 |
|---|---|
| 予算アラート | §3 のとおり。最後の砦 |
| Cloud Monitoring | Functions のエラー率・実行時間、Vertex AI のリクエスト数にアラート（週次ジョブの失敗は翌週まで気づけないため、エラー時アラートは必須） |
| structured logging | グループ単位の成功/失敗/スキップ件数を週次ジョブの最後にサマリーログ出力 |
| キルスイッチ | `system/config.suggestionsEnabled`（Firestore）。false で週次ジョブが即終了。デプロイ不要で停止可能 |
| 品質追跡 | `suggestions` ドキュメントに `model` / `inputStats` を残し、モデル更新時の品質比較に使う |

## 5. プライバシー・規約面の対応（リリース前に必須）

- 購買履歴（商品名・購入日時）を Vertex AI に送信することを
  **プライバシーポリシーに明記**する。Vertex AI はデフォルトで顧客データをモデル学習に
  使用しない方針だが、ユーザーへの開示義務とは別問題。
- 送信データの最小化: `note` / `imageUrl` / 表示名は送らない（[01](./01-履歴データ設計.md) §2.1
  でそもそも履歴に保存しない）。uid はプロンプトに含めない。
- 提案画面に「AI による提案」であることを明示（[03](./03-提案の表示と通知設計.md) §1.3）。
- 将来的にオプトアウト（グループ単位で AI 提案を無効化する設定）を提供するかは
  [05](./05-未解決論点と決定ログ.md) の論点とする。
