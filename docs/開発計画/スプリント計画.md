### 【技術基盤】確定構成：Flutter + Docker + Firebase Hosting

> **決定日：** 2026-04-19  
> **背景：** デバッグ・動作確認の高速化のため、Web ブラウザからアクセスできる構成を採用。開発環境はホスト非依存とするため Docker に統一する。  
> **決定：** Flutter（Dart）+ flutter_riverpod + go_router + FlutterFire を技術基盤とし、Web ターゲットを `firebase deploy --only hosting` で Firebase Hosting に配信する。

#### ✅ 採用構成

**前提条件：**
- Firebase は **FlutterFire**（`firebase_core` / `firebase_auth` / `cloud_firestore` / `firebase_storage`）を使う。Web / iOS / Android を単一 API で抽象化する
- Firebase 設定は `lib/firebase_options.dart`（`flutterfire configure` で生成）に集約する
- UI は Flutter の Widget ツリー（`Scaffold` / `Text` / `ListView` 等）で構築する
- 状態管理 / DI は flutter_riverpod、ルーティングは go_router（認証ガード付き宣言的ルーティング）
- i18n は easy_localization（`assets/translations/ja.json` / `en.json`、`'key'.tr()`）
- 開発環境は Docker に統一し、Flutter / Firebase CLI をコンテナ内で実行する（ホストへの個別インストール不要）

#### React Native（Expo）版からの移植マッピング（参考）

| React Native（Expo）版 | Flutter 版 |
|---|---|
| Expo Router（`app/` ファイルベース） | go_router（`presentation/router/app_router.dart` 宣言的） |
| React Context / hooks（`AuthContext` / `useItems`） | flutter_riverpod プロバイダ（`auth_providers.dart` / `item_providers.dart`） |
| Firebase JS SDK / React Native Firebase | FlutterFire（`firebase_auth` / `cloud_firestore` / `firebase_storage`） |
| `onSnapshot` | Firestore `snapshots()`（`Stream` + `StreamProvider`） |
| i18next + expo-localization | easy_localization |
| StyleSheet / `View` / `Text` | Flutter Widget（`Container` / `Column` / `Text` 等） |
| Vercel（`npx expo export --platform web`） | Firebase Hosting（`flutter build web` → `firebase deploy --only hosting`） |
| `.env`（`EXPO_PUBLIC_*`） | `lib/firebase_options.dart` + `--dart-define`（`APP_URL` 等） |

#### Web デプロイ設定（Firebase Hosting）

```bash
# Web リリースビルド（build/web/ に出力）
docker compose run --rm flutter flutter build web --release

# Firebase Hosting へデプロイ
docker compose run --rm flutter firebase deploy --only hosting
```

- ホスティング設定は `firebase.json`（`public: build/web`、SPA フォールバック）で管理する。
- Web 単一ページアプリのルーティングは go_router が担う。

#### 既知の制約・注意点

| 項目 | 内容 |
|---|---|
| **ターゲット** | 当面は Web ターゲットを優先検証。Android は Android SDK、iOS は macOS + Xcode が別途必要で未検証（`docs/REMAINING_TASKS.md` 参照） |
| **Web でのプッシュ通知** | FCM Web は Service Worker が必要。iOS Safari は Web Push の対応が限定的。現状は通知フラグの保存のみで FCM 連携は将来実装 |
| **オフライン永続化** | `cloud_firestore` が Web（IndexedDB）/ Native（SQLite）でローカルキャッシュ + 自動同期を標準提供。アプリ層の有効化処理は不要 |
| **ウィジェットテストの i18n** | easy_localization を使うテストは `test/helpers/test_localization.dart` の同期アセットローダーを使う（非同期ローダーは複数テストで空描画になる既知問題） |

## 完了の定義（Definition of Done）

以下をすべて満たしたストーリーを「完了」とみなす。

- [ ] ストーリーのすべての Acceptance Criteria を満たしている
- [ ] `flutter analyze` がエラーなしで通過する
- [ ] `dart format --output=none --set-exit-if-changed lib test` が通過する（フォーマット済み）
- [ ] `flutter test` が通過する（該当する単体 / ウィジェットテストを追加・更新済み）
- [ ] Web ブラウザで動作確認済み（Android / iOS は環境依存のため別途検証）
- [ ] UI文言がすべて `.tr()`（easy_localization）経由（ハードコード禁止）

## リスクとエスカレーション基準

| リスク | 発生条件 | 対応 |
|---|---|---|
| **Sprint遅延** | 1 Sprint で計画SP の50%未満しか完了しない | SM がスコープ調整。S03-04（通知）をPhase 2へ移動する選択肢を検討 |
| **オフライン同期バグ** | S01-04テストで同期失敗が再現する | Sprint 6 を延長。S01-06（コンフリクト解決）のPhase 3入りを確認 |
| **クローズドβ ゲート未達** | P0バグあり or クラッシュ発生 | Sprint 8 を1週間延長（最大1回） |
| **オープンβ リテンション不足** | 1週間リテンション < 40% | GA延期。オンボーディング緊急修正（2週間） |
| **Firebase 費用スパイク** | 想定費用の2倍超 | Firestoreルール最適化 + 読み取り上限アラート設定を即実施 |
| **環境構築・移行遅延** | Sprint 0 で Docker ビルド / Web 起動の疎通が取れない | Docker ビルド → `flutter run -d web-server`（`localhost:5000`）→ Firebase Hosting 疎通確認を Sprint 0 完了の追加条件とすることで判断先送りを防ぐ |
