# エラー仕様書

このドキュメントはアプリ内で発生するエラーの分類・発生条件・エラーコード・表示メッセージを定義します。
実装時はこのドキュメントを参照し、エラーコードの追加・変更時は必ずここを更新してください。

UI 層は Firebase 固有の例外（`FirebaseException` / `FirebaseAuthException`）に直接依存しません。Repository 層（`lib/data/repositories/`）の catch で `toAppError()`（`lib/data/firebase/firebase_error_converter.dart`）により `AppError`（`lib/core/errors/app_error.dart`）へ変換し、UI は `e is AppError` で型チェックして `e.code`（`AppErrorCode` enum）で分岐します。

## 表示方法の種類

| 種別 | 説明 | 使いどころ |
|---|---|---|
| インライン（フォーム内） | フォーム上部に `MessageBox`（`lib/presentation/widgets/message_box.dart`）で表示 | 入力値に起因するエラー（バリデーション、認証） |
| トースト | `AppFeedback.showToast(context, msg, type: ToastType.error)`（`lib/presentation/widgets/app_feedback.dart`、`ScaffoldMessenger` ベース）で画面下部に一時表示 | 非同期操作の失敗（Firestore 書き込みなど） |

文言はすべて `easy_localization` の `'key'.tr()` 経由で扱う（ハードコード禁止）。認証エラーの文言変換は `lib/presentation/utils/error_messages.dart`（`loginErrorMessage` / `signupErrorMessage`）に集約しています。

## 1. 認証エラー（AppError 経由）

```dart
import '../../core/errors/app_error.dart';
import '../utils/error_messages.dart';

try {
  await ref.read(authControllerProvider).signup(email, password);
} catch (e) {
  // signupErrorMessage が AppErrorCode を見て i18n キーを解決する
  setState(() => _error = signupErrorMessage(e));
}
```

`signupErrorMessage` / `loginErrorMessage` の内部は `AppErrorCode` の `switch` で、未マップは汎用キー（`signup_failed` / `login_failed`）にフォールバックします。

### サインアップ時

| `AppErrorCode` | 発生条件 | i18n キー | 表示方法 |
|---|---|---|---|
| `authEmailAlreadyInUse`（`auth/email-already-in-use`） | 登録済みのメールアドレスで作成しようとした | `auth.error.email_already_in_use` | インライン |
| `authWeakPassword`（`auth/weak-password`） | パスワードが 6 文字未満 | `auth.error.weak_password` | インライン |
| `authInvalidEmail`（`auth/invalid-email`） | メールアドレスの形式が不正 | `auth.error.invalid_email` | インライン |
| その他 | 上記以外（`authUnknown` / `authNetworkRequestFailed` など） | `auth.error.signup_failed` | インライン |

### ログイン時

| `AppErrorCode` | 発生条件 | i18n キー | 表示方法 |
|---|---|---|---|
| `authInvalidCredential`（`auth/invalid-credential`） | メール or パスワードが一致しない（統合コード） | `auth.error.invalid_credential` | インライン |
| `authTooManyRequests`（`auth/too-many-requests`） | 短時間に多数のログイン失敗でアカウントが一時ロック | `auth.error.too_many_requests` | インライン |
| その他 | 上記以外 | `auth.error.login_failed` | インライン |

> **注意**: Firebase Auth では `user-not-found` / `wrong-password` は `invalid-credential` に統合された。`toAppError()` では旧コード（`user-not-found` / `wrong-password`）も `authInvalidCredential` に寄せる。FlutterFire の `FirebaseAuthException.code` は `auth/` プレフィックスを持たない（例: `invalid-credential`）ため、`toAppError()` は先頭の `auth/` を除去してからマッピングする。

## 2. データアクセスエラー（Firestore 由来）

Firestore の読み書き失敗は Repository 層で `AppError`（`data/*` プレフィックス）に変換されてから UI に伝播する。UI はトーストで通知し、操作の種別ごとに文言を分ける。

```dart
try {
  await ref.read(itemRepositoryProvider).addItem(groupId, draft, order);
} catch (e) {
  AppFeedback.showToast(context, 'app.error.add'.tr(), type: ToastType.error);
}
```

| 操作 | 発生条件 | i18n キー | 表示方法 |
|---|---|---|---|
| データ取得失敗 | `watchItems` の `Stream` onError（`AsyncValue.error`） | `app.error.fetch` | トースト |
| アイテム追加失敗 | `addItem` の失敗 | `app.error.add` | トースト |
| アイテム更新失敗 | `setVolunteer` / `setPurchased` / `updateItemDetails` の失敗 | `app.error.update` | トースト |
| アイテム削除失敗 | `deleteItem` の失敗 | `app.error.delete` | トースト |
| ユーザードキュメント作成失敗 | サインアップ後の `createUserDocument` 失敗 | `auth.error.signup_failed` | インライン |

### `DataErrorCode` と Firestore エラーコードの対応

| `AppErrorCode`（= データレイヤー） | Firestore error code | 意味 |
|---|---|---|
| `dataPermissionDenied`（`data/permission-denied`） | `permission-denied` | セキュリティルールで拒否 |
| `dataNotFound`（`data/not-found`） | `not-found` | ドキュメントが存在しない |
| `dataUnavailable`（`data/unavailable`） | `unavailable` | ネットワーク不通でオフライン |
| `dataResourceExhausted`（`data/resource-exhausted`） | `resource-exhausted` | クォータ超過 |
| `dataTagLimitExceeded`（`data/tag-limit-exceeded`） | なし（`GroupController` 層でスロー） | タグ数がプラン上限に達した |
| `dataUnknown`（`data/unknown`） | 上記以外（非 `FirebaseException` 例外含む） | 未マップのエラー |

## 3. グループ管理エラー（ドメイン）

Firebase を呼ぶ前に `GroupController` / `AuthController`（`lib/presentation/providers/`）でビジネスルールを検証し、直接 `AppError` を throw する。`toAppError()` は不要。

| `AppErrorCode` | 発生条件 | i18n キー | 表示方法 |
|---|---|---|---|
| `groupOwnerCannotLeave`（`group/owner-cannot-leave`） | オーナーが脱退しようとした | `group.settings.error.owner_cannot_leave` | インライン |
| `groupCannotRemoveOwner`（`group/cannot-remove-owner`） | オーナーを強制退場させようとした | `group.settings.error.cannot_remove_owner` | インライン |
| `authCannotDeleteOwner`（`auth/cannot-delete-owner`） | オーナーのまま退会しようとした | `profile.error.cannot_delete_owner` | インライン |

```dart
// Controller 層での throw 例（toAppError() は不要）
if (target.ownerId == uid) {
  throw const AppError(
    AppErrorCode.groupOwnerCannotLeave,
    'Owner must disband or transfer',
  );
}
```

## 4. バリデーションエラー（クライアント）

Firebase を呼ぶ前にクライアントで検証し、即時インライン表示する。

| 発生条件 | i18n キー | 表示方法 |
|---|---|---|
| パスワードが空 | `auth.error.empty_password` | インライン |
| パスワードと確認用パスワードが不一致 | `auth.error.passwords_do_not_match` | インライン |
| グループ名が空 | `group.settings.error.empty_name` / `group.create.error.empty_name` | インライン |
| アイテム名が空 | `item.name_required` | インライン |
| 招待コードが空 | `group.join.error.empty_code` | インライン |

## 6. `AppErrorCode` と Firebase error code の対応表

`toAppError()`（`lib/data/firebase/firebase_error_converter.dart`）が以下のマッピングで変換する。

### 認証レイヤー

| `AppErrorCode` | Firebase Auth error code（`auth/` プレフィックス除去後） |
|---|---|
| `authEmailAlreadyInUse` | `email-already-in-use` |
| `authWeakPassword` | `weak-password` |
| `authInvalidEmail` | `invalid-email` |
| `authInvalidCredential` | `invalid-credential`、`user-not-found`、`wrong-password` |
| `authTooManyRequests` | `too-many-requests` |
| `authNetworkRequestFailed` | `network-request-failed` |
| `authUnknown` | 上記以外の `auth/*` |

### データアクセスレイヤー

| `AppErrorCode` | Firestore error code |
|---|---|
| `dataPermissionDenied` | `permission-denied` |
| `dataNotFound` | `not-found` |
| `dataUnavailable` | `unavailable` |
| `dataResourceExhausted` | `resource-exhausted` |
| `dataTagLimitExceeded` | なし（`GroupController` 層の直接 throw） |
| `dataUnknown` | それ以外（非 `FirebaseException` 例外含む） |

## 7. 更新ルール

- 新しいエラーを実装した場合は、必ずこのドキュメントの該当セクションに行を追加すること。
- i18n キーを追加した場合は「5. i18n キーと文言の対応表」と `assets/translations/ja.json` / `en.json` を同時に更新すること。
- `AppErrorCode` enum の拡張時は「6. `AppErrorCode` と Firebase error code の対応表」と
  `lib/core/errors/app_error.dart`、`lib/data/firebase/firebase_error_converter.dart` を同時に更新する。
- FlutterFire / Firebase SDK のバージョンアップにより error code が変わった場合は、このドキュメントおよび
  `firebase_error_converter.dart` を両方更新すること。
